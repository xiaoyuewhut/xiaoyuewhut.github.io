# 感知类辅助功能 Perception Aids
这些主要是“告诉驾驶员情况”，不主动控制车辆。其实更侧重于对前向的感知，全向也有。

+ **FCW — Forward Collision Warning 前向碰撞预警**
+ PCW — Pedestrian Collision Warning 行人碰撞预警
+ AEB-Warning — 自动紧急制动警告
+ LDW — Lane Departure Warning 车道偏离预警
+ BSD — Blind Spot Detection 盲区监测
+ BSW — Blind Spot Warning 盲区预警
+ LCA-W — Lane Change Assist Warning 换道辅助预警
+ RCW — Rear Collision Warning 后碰撞预警
+ CTA — Cross Traffic Alert 横向来车预警（R-CTA / F-CTA）
+ IHC / AHB — Intelligent High Beam / Auto High Beam 自动远光切换
+ TSR — Traffic Sign Recognition 交通标志识别
+ DAW — Driver Attention Warning 驾驶员注意力预警

# 纵向控制类 Longitudinal Control
车辆“前后方向速度与距离”控制。
## 基础级
+ **CC — Cruise Control 定速巡航**
+ LSF — Low Speed Follow 低速跟随

## 中高级
+ **ACC — Adaptive Cruise Control 自适应巡航**

+ iACC / eACC — Intelligent ACC / Enhanced ACC（结合导航/地图）

+ TJA-L（纵向部分） — Traffic Jam Assist 交通拥堵跟车（含自动启停）

# 横向控制类 Lateral Control
车辆“方向盘”控制。

+ **LKA — Lane Keeping Assist 车道保持辅助（小幅度纠偏）**

LCK — Lane Keeping Control（比 LKA 更主动保持）

LCA — Lane Centering Assist 车道居中保持（连续控制）

ALC（Lateral 部分） — Auto Lane Change 自动换道辅助

# 横纵向组合控制类
## 高速场景

+ **LCC — Lane Centering Control（横纵向组合的车道居中）**

+ HWA — Highway Assist 高速公路辅助

+ HWP — Highway Pilot 高速 pilot（接近自动驾驶但常需人监督）

+ **NOA — Navigate on Autopilot/Navigation Assisted Driving
（高速 + 匝道 + 出入口 + 自动换道 + 导航跟随）**

## 城市场景

+ U-NOA / C-NOA — Urban/City Navigate on Autopilot

+ ALC — Auto Lane Change 自动换道

+ ILC — Intelligent Lane Change 智能换道

+ LCA（Decision版） — Lane Change Assist 决策式换道辅助

+ ELK — Emergency Lane Keeping 紧急车道保持

## 拥堵 & 巡航场景

+ TJA — Traffic Jam Assist 交通拥堵辅助

+ ICA — Integrated Cruise Assist 综合巡航辅助

+ SCA — Steering Control Assist 转向控制辅助

# 问题
## ACC、LCC、NOA有什么区别？
ACC无法自主变道，无法配合红绿灯自动启停。组合拨杆向下拨一次。

LCC在ACC的基础上增加了车道保持功能，打方向灯即可自动变道，可以配合红绿灯自动启停。未开启导航的情况下，组合拨杆向下拨两次。

NOA分为高速NOA和城区NOA，结合导航信息，自动完成变道、超车等。开启导航的情况下组合拨杆向下两次。

## 高速NOA和城区NOA的区别？
高速 NOA 的场景**高结构化、参与者简单、变量少、规则明确。**
但是 **城区 = 非结构化、参与者杂、规则复杂、交互多，是自动驾驶真正的难点所在**。城区NOA本质上是一套接近 L3 的完整自动驾驶系统。

| 项目           | 高速 NOA     | 城区 NOA                 |
| ------------ | ---------- | ---------------------- |
| **场景类型**     | 高结构化       | 非结构化                   |
| **参与者**      | 汽车为主       | 汽车＋行人＋自行车＋电动车＋动物＋乱停障碍… |
| **速度**       | 高速但行为简单    | 低速但行为复杂、交互多            |
| **车道线**      | 清晰、稳定      | 缺失/模糊/遮挡               |
| **交通元素**     | 无红绿灯/无路口   | 大量路口、红绿灯、斑马线           |
| **HD Map依赖** | 中等         | 非常强                    |
| **感知难度**     | 中          | 极高                     |
| **预测复杂度**    | 低          | 极高                     |
| **决策逻辑**     | 模板稳定       | 交互式、高不确定性              |
| **规划难度**     | 低          | 高                      |
| **算力需求**     | 中（几十 TOPS） | 高（100–400 TOPS）        |
| **软件复杂度**    | L2 / L2+   | L2++ / 近 L3            |
| **安全边界**     | 清晰         | 难界定，需大量 case           |
